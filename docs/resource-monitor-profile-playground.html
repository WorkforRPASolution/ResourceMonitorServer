🖥️ 설비 모니터링 설정 가이드 & Playground

코딩을 몰라도 괜찮습니다. 아래 4단계를 채우면 우측에 실제 모니터링 설정(RESOURCE_MONITOR_PROFILE)이 자동으로 만들어집니다.

1 누구에게 (scope)
→
2 무엇을 잰다 (measures)
→
3 언제 경보 (rules)
→
4 어떻게 알림 (notify)
빠른 예시:
1

누구에게 적용할까요? scope

🎯 "누구에게" — 이 설정을 어느 장비에 적용할지 정합니다. 병원에서 "전체 환자" 또는 "특정 환자 1명"에게 처방하듯, * 는 전체를 뜻합니다. 잘 모르겠으면 그대로 두세요(전체 장비 적용).
예) 특정 설비만: 공정 PHOTO, 설비 EQP001. 전체 적용은 모두 *.
2

무엇을 잴까요? measures

🌡️ "잰다" — 어떤 온도계를 어디에 꽂을지 고르는 단계입니다. 측정 항목(예: CPU 사용률)을 정하고, 최근 몇 분(window)을 볼지, 그 구간에서 어떤 값(최댓값·평균 등)을 뽑을지 정합니다.
시스템 전체는 @system. 특정 프로그램이면 프로그램명.
최근 몇 분을 모아서 볼지.
최댓값(max)·최솟값(min)·평균(avg)·상위값(p95)·급증횟수(spike_count) 중 선택.
3

언제 경보할까요? rules

🚨 "판단한다" — 잰 값이 어떤 선을 넘으면 경보를 울릴지 정하는 단계입니다. "체온이 38도를 넘으면 알림"처럼, 측정값(2단계의 값) + 비교 + 기준 숫자로 조건을 만듭니다. 조건을 2개 이상 넣으면 모두 만족(AND) 또는 하나라도 만족(OR)을 고를 수 있어요.
몇 분마다 검사할지.
예) cpu.max 가 >= 80 이면 경보.
4

어떻게 알릴까요? notify

📧 "알린다" — 경보가 발생하면 어떻게 메일을 보낼지 정합니다. 같은 경보가 계속 울리면 메일 폭탄이 되니, 재알림 간격(cooldown)을 두어 "한 번 알린 뒤 N분 동안은 다시 안 보냄"으로 조용히 만듭니다.
한 번 알린 뒤 이 시간 동안은 같은 경보를 다시 안 보냅니다.
